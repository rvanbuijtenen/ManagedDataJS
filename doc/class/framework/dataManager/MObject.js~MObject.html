    MObject

    MObject implements the basic functionality for managed data:

    It provides a set() and get() method for managed fields that will be invoked by the proxy wrapped around this object. Managed fields are stored in the 'data' property of an MObject.

    Furthermore, it provides a notifyArray() callback that will be invoked whenever a method on an ArrayMField contained in the MObject's fields is called

    public beforeArray(method: String, args: Array, array: ArrayMField): Array source

    Methods invoked on managed array fields cannot be trapped by this MObjects proxy, therefore the array invokes the notifyArray callback when one of it's methods are invoked. This allows the managed object and possible mixins to perform reflective behaviour on arrays, as wel as regular data.

    An array containing the arguments given to the array's method
    array ArrayMField

    The array that invoked the callback

    Return:

    Array

    The arguments given to notifyArray. Mixins can override this method to intercept and possibly modify the arguments.
