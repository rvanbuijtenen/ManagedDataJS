    MObject

    MObject implements the basic functionality for managed data:

    It provides a set() and get() method for managed fields that will be invoked by the proxy wrapped around this object. Managed fields are stored in the 'data' property of an MObject.

    Furthermore, it provides a notifyArrayChanged() callback that will be invoekd whenever an ArrayMField changes its state.

    public notifyBeforeArrayChanged(array: ArrayMField): Symbol source

    By default the modification methods of managed arrays bypasses the set trap. Because of this, arrays notify their superKlass when they change. This way an MObject can still implement its functionality whenever an array field is modified by extending this function.

    public setThisProxy(proxy: Proxy) source

    A MObject needs to know its own proxy, since we need to pass a proxied reference of the MObject that an MObjectMField belongs to, and non-proxied MObjects may never be interacted with except through 'this'. So in order to ensure proper inverse relations, and MObjectMField must have a proxied reference to the object that it belongs to. Setting the proxy must be done by the factory function.

    Params:

    NameTypeAttributeDescription
    proxy Proxy

    The proxy wrapped around this MObject

    Throw:

    TypeError

    The given proxy must be the proxy wrapped around this, otherwise an error is thrown
